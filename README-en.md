# WinEzCtrlKit

A Win32 common utilities library.

* Built with C++20
* Maintains a global list of all windows and provides various automation facilities, including high-DPI awareness and dark/light mode switching
* Provides four layout types: Linear, Flow, Grid, and Frame
* Adds an additional layer on top of `CWindow::Create` and `CLayoutBase`, enabling declarative-style control creation
* Offers simple two-way data binding through `Observable<>` and `DdxBind`-prefixed functions

  ```C++
  Observable<CStringW> obString{};
  DdxBindEdit(m_Edit, *this/* parent */, obString);
  ```
* Uses `OnMessage` instead of a traditional window procedure. Additionally, the following mechanisms are provided to customize message handling:

  * Message loop filtering: supports `IsDialogMessageW`, accelerator translation, keyboard filtering, etc.
  * Self-notification handling: `CWindow::OnNotifyMessage` receives all notifications generated by the window itself
  * Window message filtering: `CWindow::GetEventChain` returns an event chain object that stores a sequence of callbacks. When a message is generated, each callback in the chain is invoked in order before `OnMessage`. A callback may intercept a message and prevent it from propagating further
* Provides full encapsulation of most standard controls, with necessary adaptations for dark mode support
* Does not abstract specific messages (such as button clicks or window resize events). The traditional message-handling workflow is preserved, and most operations directly use the original Win32 API
